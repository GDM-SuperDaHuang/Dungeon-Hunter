# 常用方法
## Object Type
1) 定义：Object Type（物体类型）是每个有碰撞的Primitive Component（如Static Mesh、Skeletal Mesh、Capsule等）的一个身份标签，它决定这个物体在碰撞系统中“属于哪一类”。
作用：
它属于Object Channels（物体通道）中的一个具体通道。
当两个物体发生碰撞或查询时，引擎会查看：一方是“谁”（Object Type），另一方对这个类型有何响应（Block/Overlap/Ignore）。

常见例子：
一个墙壁的Static Mesh Component → Object Type = WorldStatic。
一个玩家角色的Capsule Component → Object Type = Pawn。
一个可推动箱子 → Object Type = PhysicsBody 或 WorldDynamic。

## Trace Channels
1) Trace Channels（追踪通道）是专门用于射线追踪（Line Trace）、形状追踪（Sweep Trace）、重叠检查等“查询操作”的通道。
